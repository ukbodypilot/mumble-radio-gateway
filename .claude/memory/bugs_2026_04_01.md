## 2026-04-01 Bug Log

**False watchdog → PortAudio segfault (3 crashes in 24h):**
`check_watchdog()` used `gateway.last_audio_capture_time` which is only updated by the main thread's primary listen bus mixer. When AIOC was routed to a solo bus (BusManager thread), the timestamp never updated. Watchdog falsely concluded "no audio for 10s" and called `_restart_audio_input()` every few seconds. The restart closed the PortAudio stream while `_rx_reader_loop` was blocked inside `read()` on another thread → libportaudio segfault.

**Fix:** Use plugin-local `_last_audio_capture_time` (set by reader thread on every successful read) instead of gateway global. Also: set `_input_stream = None` before closing to let reader exit the blocking read safely; capture local stream ref in reader loop.

**XRUN at startup → permanently dead audio:**
`_init_audio_streams()` opened the ALSA capture stream with `start=True` (default), but the reader thread didn't start until after CAT and relay init (~1s later). The ALSA ring buffer overflowed (`appl_ptr: 0`, `state: XRUN`) before the first `read()`. PortAudio's blocking read on an XRUNed stream either hung or silently failed forever. No error logging in the exception handlers meant this was completely invisible.

**Fix:** Open stream with `start=False`, call `start_stream()` at the top of `_rx_reader_loop` when reader is ready to consume. Added logging to all exception handlers in the reader loop.
