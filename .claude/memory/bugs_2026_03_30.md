## 2026-03-30 Bug Log

**json vs json_mod:** `_source_on_listen_bus()` used `json.load()` but gateway_core.py imports json as `json_mod`. NameError silently caught by `except Exception: pass` — ALL sources fell through to "routed via bus manager" even when on listen bus. SDR had no audio for an entire session.

**Double PCM push:** Main loop pushed PCM at two points per tick (after mixer.tick AND before VAD gate). Caused 50/50 choppy audio on listen bus PCM.

**Hardcoded 'listen' bus ID:** All `_bus_sinks.get('listen', ...)` and `_bus_stream_flags.get('listen', ...)` hardcoded bus ID 'listen'. User renamed listen bus to 'sdr_mix' — all gates failed silently. Replaced with `self._listen_bus_id` resolved from routing config.

**TH-9800 not in _get_radio_plugin:** `aioc_tx`/`aioc` had no mapping to `th9800_plugin` in BusManager. Solo bus couldn't find its radio.

**TH-9800 source mapped to dead object:** `_get_source('aioc')` returned `gw.radio_source` (old AIOCRadioSource) instead of `gw.th9800_plugin`.

**Queue competition:** TH-9800 added to both primary ListenBus AND solo bus — both called `get_audio()`, splitting the audio queue 50/50.

**TX sink set as radio:** `aioc_tx` bus-sink resolved as SoloBus radio, causing `get_audio()` on a TX-only path. Added `_tx_only` flag.

**VAD gating broadcastify:** Broadcastify delivery was after VAD gate — SDR scanner audio (below VAD threshold) never reached it. Moved to early delivery path.

**VAD gating Mumble TX:** Same issue — Mumble sink delivery after VAD gate. Moved to early delivery alongside broadcastify.

**Mumble TX level showing RX:** Mumble sink level bar read `rx_audio_level` (incoming) instead of tracking outgoing audio. Added `mumble_tx_level`.

**P/M button default logic:** `proc.pcm !== false` treated `undefined` as truthy — buttons showed ON when no processing config existed. Changed to `proc.pcm === true`.

**SDR pgrep -x:** Changed `pgrep -f` to `pgrep -x` for rtl_airband — binary name didn't match exactly when daemonized. Reverted to plain `pgrep`.

**sdrplay service "active" but dead:** Service reported active but SoapySDR couldn't find device. Added sdrplay restart to SDR retry loop.

**DarkIce feeding garbage:** DarkIce ran independently of routing, sending silence/garbage to Broadcastify when nothing fed it. Replaced entire external chain with direct Icecast streaming inside gateway.

**Old sound_received_handler conflict:** Direct `output_stream.write()` + PTT management in Mumble callback conflicted with SoloBus path. Disabled old direct path — bus handles PTT and TX audio now.
